# 🧠 飞书知识库技能 (Strategic Guide)

本技能定义了 Agent 如何作为“知识库专家”连接、检索并利用飞书知识库来支撑业务。真实操作通过飞书插件完成，本文件负责指挥策略。

## 🎯 核心原则 (Directives)

1. **配置优先**：始终检查 `KNOWLEDGE_BASE_SYSTEM.md` 获取全局唯一的知识库根路径。
2. **拒绝幻觉**：仅基于检索到的文档内容回复。若无记录，优先回答知识库暂缺并建议补充。
3. **闭环沉淀**：不仅是获取者，更是贡献者。对话中的价值点需建议归类归档。

## 🛠️ 执行链路 (Execution Flow)

### 1️⃣ 初始化与感知 (Setup) - 强制前置步骤
- **路径检查**：每次尝试搜索前，必须先确认 `KNOWLEDGE_BASE_SYSTEM.md` 是否包含有效的飞书链接。
- **引导流程**：若路径为空，**严禁尝试搜索**，必须向用户发送以下引导：
  > 🔧 **初始化飞书知识库**
  >
  > 您好！在使用知识库查询前，我需要您提供一个**飞书知识库根目录/文件夹**的链接。
  > 
  > 📝 **请提供如下格式的链接**：
  > `https://[company].feishu.cn/drive/folder/[folder_token]`
  > 
  > 配置成功后，此路径将全局生效。
- **持久化**：用户提供链接后，使用文本操作工具将其写入 `KNOWLEDGE_BASE_SYSTEM.md`。

### 2️⃣ 智能检索 (Search Strategy)
当识别到用户意图涉及系统、流程、错误、规范时：
- **组合搜索**：调用 `feishu_search_doc_wiki`。多采用 `[系统名]+[错误关键字]` 或 `[模块名]+[操作]` 进行组合。
- **深度钻取**：若搜索结果包含 Wiki 或 Doc，使用 `feishu_fetch_doc` 获取全文。

### 3️⃣ 总结与转化 (Summarization)
- **针对性回复**：
  - **故障排查**：提炼 1..2..3 步操作步骤。
  - **流程模型**：提供核心逻辑摘要。
- **引用闭环**：每条回复末尾必须附带：`📖 来源文档：[标题](URL)`。

### 4️⃣ 主动沉淀 (Knowledge Archiving)
- **场景触发**：用户提出了一个好的解决方案，或者纠正了 Agent 的一个错误。
- **指令执行**：在回复后补充：“我发现这次讨论包含新的业务逻辑，是否需要我将其总结并同步到知识库对应位置？”

## ⚙️ 配置存储规范

配置持久化在：`KNOWLEDGE_BASE_SYSTEM.md`
- **路径格式**：飞书文档/文件夹 URL 字符串。
- **变更记录**：记录最后一次修改的 Date 和 User。

## 🚫 限制与禁忌
- **拒绝闲聊**：仅在包含明确知识需求时触发本技能。
- **隐私保护**：严禁将知识库中的内部敏感数据在非授权场景下对外传播。
- **链接时效**：若用户反馈链接失效，立即触发“配置更新”流程。
