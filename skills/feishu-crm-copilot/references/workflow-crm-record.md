# 业务插件：跟进记录填报指南

本脚本已由 `SKILL.md`基座成功挂载。专门用于将原始沟通素材专业转化为高质量、商业洞察驱动的记录。

---

## ⚙️ 核心逻辑要求 (Core Logic)

### 1. 商业洞察提取与重塑 (Extraction Strategy)
- **按骨架重组**：**必须**参照 `高质量示例.txt` 的标准结构来填充内容（例：分为“1.客户长期需求”、“2.需求进展”、“3.客户疑问与答疑记录”）。
- **拒绝流水账摘要**：不要用“会议概览”、“小结：1. 2. 3.”。
- **依据质检自评**：提取草稿后，对照 `质检标准.txt`（包含客户有效信息50分、明确下一步30分、清晰结构化20分），给草稿进行量化自评。
- **内容区分边界**：【跟进记录】中写“已发生事实与价值结论”，【下一步跟进计划】只写“待办的具体 action”，**严禁相互复述杂糅**。

### 2. 👁️ 图像防幻觉与深度扫描 (Deep Scan)
- **穷举扫描协议**：对聊天截图、会议记录图片执行**全覆盖式扫描**。捕捉包括页眉标题、小字备注、URL 后缀等所有实体信息。
- **禁止脑补**：严禁仅凭邮箱域名猜测公司主体。若识别出多个公司主体，必须将【客户简称】置为“待补充”，并列出候选项。

---

## ⚙️ 字段提取与预览规范

### 1. 字段映射目标
- 🔴 **必填 (Mandatory Fields)**：
    1. **【客户简称】**：公司/主体名称 (严禁填入人名)。
    2. **【沟通对象】**：具体的对接人姓名/职务 (支持复数)。
    3. **【跟进记录】**：呈现事实与结论。
    4. **【下一步跟进计划】**：具体的 action。
- ⚪ **选填 (Optional Field)**：
    5. **【内部参与人】**：协助跟进的人员。

### 2. 自动化质检交互 (QA Menu)
- **交互流程**：在预览列表下方，强制输出质检得分、优化建议以及确认菜单 `[1][2][3]`。

### 3. 数据关联与校验
- **ID 前置匹配**：预览前必须执行 `客户表` ID 查找。若 `客户简称` 匹配失败，即刻执行拦截禁止入库动作。

---

## 🚀 入库执行规范
- **写入目标**：`FEISHU_CRM_RECORD_TABLE_ID`。
- **格式终结清洗**：入库动作触发前，基座需物理剥离所有的 Markdown 语法标记 (`**`, `#`, `*`)，确保入库文本整洁。
- **单回合物理断点**：预览、自评与写入动作必须被用户确认口令锁死物理隔离。
