# WeWe-RSS 公众号订阅与查询工具 (wewe-rss-add-feed) v1.0.2

这个 Skill 专门用于业务人员自动化添加与查询微信公众号到本地蜘蛛网监控库（`wewe-rss` 容器）中，支持智能分流与防风控保护。

## **使用场景**
当业务人员在群里或者私信发来需求：
1. **添加监控**（带有历史文章链接）：
   - "把这个公众号加入监控"
   - "加一下订阅源"
2. **查询白名单**：
   - "我们现在监控了哪些公众号？"
   - "蜘蛛网里都有哪些源？"
   - "查一下列表"

## **工作机制 (v1.0.2 安全降级与查询策略)**
1. **自动分流（阈值 >= 3）**：
   - **单发模式 (1-2条)**：零延迟。极速解析 -> 落盘 -> 强制唤醒本地爬虫立即拉取最新文章。
   - **批量模式 (>=3条)**：静默防风控。启用 1~3 分钟的随机休眠。流式解析，边解析边入库。**不强制唤醒爬虫**，交由原生后台慢慢消化，避免微信小号被风控封锁。
2. **状态查询**：直插底层 `wewe-rss-db-1` 数据库，拉取所有当前状态为开启 (`status=1`) 的公众号名单。
3. **数据直灌**：不再依赖脆弱的浏览器端，直接在容器底层中以 `utf8mb4` 字符集写入数据，彻底杜绝乱码。

## **使用方法**
**不要通过浏览器界面。直接在终端中执行对应的 Node.js 脚本：**

- **添加订阅源**（支持传入单个或多个 URL，用空格分隔）：
  ```bash
  node ./skills/wewe-rss-add-feed/add.js "URL1" "URL2" "URL3"
  ```
- **查询订阅源列表**：
  ```bash
  node ./skills/wewe-rss-add-feed/list.js
  ```

## **执行提示**
- 对于 **添加任务**：
  - 如果是 **1~2 条**链接，直接调用 `exec` （正常等待）并返回结果。
  - 如果是 **>= 3 条**的链接，务必使用 `exec` 工具的 `background: true` 参数转入后台执行（后台守护进程模式），并向业务人员发送一段通俗易懂的解释，例如：“已收到批量链接！微信防封控比较严格，一次性加太多号容易导致咱们的监控小号被封。为了安全起见，我已经把这批账号放进了后台的‘慢速排队通道’。系统会模拟真人，隔几分钟加一个，慢慢帮你全部搞定。加完后它们会自动开始抓取，不用你再操心啦！”
- 对于 **查询任务**：执行 `list.js`，将返回的列表直接反馈给业务人员。
