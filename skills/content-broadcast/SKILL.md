# Content Broadcast (内容解析与定向分发工作流)

## 描述
自动抓取外链文章生成摘要，并以机器人(Bot)身份推送到指定飞书群。为了适配所有模型（包括小参数模型），请严格按照以下【三步走】的死指令执行，不可跳步！

## 第一步：抓取与总结（收到链接时）
1. 优先使用 `web_fetch` 抓取链接。
2. 微信公众号(mp.weixin.qq.com) 或 fetch 失败时，必须使用 `browser` 工具：先 `action="navigate"`, 然后 `action="snapshot"`, 获取内容后务必 `action="close"`。
3. 如果彻底抓取不到，回复：“无法获取内容，请提供文字”。绝不能靠标题瞎编！
4. 读取 `memory/user-preferences.md`，按以下格式输出摘要草稿：
   - **标题与链接**：[文章原名](URL)
   - **核心事实 (What)**：通俗说明发布了什么、发生了什么。
   - **商业价值 (Why)**：对企业的降本增效、行业影响是什么。
5. 结尾必须加上这句，然后停止动作：“*草稿已生成，请审核。如需发送，请回复目标群组 ID（如 oc_xxx）。*”

## 第二步：等待用户指令（绝对静止期）
- 在用户没有给出包含 `oc_` 开头的群组 ID 并明确同意发送前，**绝对禁止**调用任何发送消息的工具。

## 第三步：定点定向发送（收到群ID时）
- 收到用户的发送指令和群 ID（例如：发到 oc_12345）。
- **唯一合法发送方式**：调用 `message` 工具。
  - 参数配置：`action="send"`, `channel="feishu"`, `target="用户给的群ID"`, `message="第一步生成的完整摘要文本"`。
- **🚫 致命红线（防呆警告）**：
  - ❌ 绝对禁止使用 `feishu_im_user_message` 工具！
  - ❌ 绝对禁止修改或缩减刚才用户确认过的摘要内容！
- 发送成功后，向用户回复：“已成功以 Bot 身份发送至群 [群ID]。”
